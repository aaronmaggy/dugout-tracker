Component Index
Every Dugout object involved in building a tracking link, and where to find it.
● Tracking Center objects
ObjectDefinitionUsed in link as
OfferA campaign set up for advertisers/clients + affiliates. Three types: Direct, Parent, Child.—
Affiliate IDUnique ID for the traffic source (affiliate). Set in Tracking Center.a=144
Creative IDUnique ID for the creative, which maps to an Offer Contract (the landing page URL).c=104294
Offer ContractThe actual landing page URL with parameters. Only active when mapped to a Creative ID.Destination URL
Marketing Campaign ID (CPID)Appended to tracking URLs for call routing. Three types: Transfer, Callback, Fallback.cpid=#s20#
SUBID (s1–s100)Predefined parameters for storing click data: keywords, ad groups, device, click IDs, etc.s1=, s2=, …
● Ring Center objects
ObjectDefinitionNotes
Dynamic RingpoolSession-based pool of phone numbers — generates a unique number per visitor session.Requires a fallback CPID
1-to-1 RingpoolSingle fixed phone number. Used for fallback numbers or affiliate hardcoded pages.No dynamic generation
Custom NativeHybrid — multiple numbers but not session-based. Assigned by predefined attributes.Rare use case
● Post Center + Datalot
ParameterDefinitionDatalot field
Post ConfigurationForwards web form data: Landing Page → Dugout → Datalot API.—
access_keyUnique value generated and maintained by the Datalot API.access_key
source_idDatalot traffic source ID — analogous to Dugout's Affiliate ID.source_id
product_idDatalot product ID — analogous to Dugout's vertical.product_id
campaign_idSource Contract ID in Datalot — analogous to Dugout's Offer ID.campaign_id
transfer_destinationTransfer phone number from the Transfer CPID — used for outbound dial transfers.transfer_destination
callback_numberCallback phone number from the Callback CPID — used when customer doesn't answer.callback_number
